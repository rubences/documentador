# 5.3 Actuación 3: Renovación de la Web de Turismo - Mantenimiento

## 5.3.1 Objeto del manual de mantenimiento

La presente sección describe los procedimientos de mantenimiento preventivo y correctivo de
la web de turismo, incluyendo actualizaciones de seguridad, optimización de rendimiento y
gestión de incidencias de disponibilidad.

## 5.3.2 Equipamiento y componentes sujetos a mantenimiento

| Componente | Tipo de mantenimiento | Frecuencia |
| --- | --- | --- |
| Plataforma web | Parches de seguridad, actualizaciones | Mensual |
| Certificado SSL/TLS | Renovación, validación | Trimestral |
| Infraestructura de caché | Limpieza, reconfiguración | Semanal |
| Base de datos de web | Optimización, integridad | Semanal |
| Plugins y extensiones | Actualización a versiones compatibles | Mensual |

## 5.3.3 Plan de mantenimiento preventivo

| Tarea | Descripción | Frecuencia | Responsable |
| --- | --- | --- | --- |
| Monitorización de disponibilidad | Verificar web accesible 24/7 | Continua | Monitorización automática |
| Auditoría de seguridad | Escanear vulnerabilidades conocidas | Mensual | Equipo seguridad |
| Validación de enlaces | Revisar integridad de vínculos internos | Quincenal | Script automatizado |
| Auditoría de rendimiento | Analizar tiempos de carga y optimización | Semanal | Equipo técnico |
| Backup de web | Realizar copia de seguridad completa | Diaria | Automatizado |

## 5.3.4 Procedimiento de actualización de seguridad

| Paso | Instrucción |
| --- | --- |
| 1 | Recibir alerta de parche de seguridad. |
| 2 | Evaluar criticidad y tiempo de aplicación disponible. |
| 3 | Aplicar en entorno de prueba y validar funcionalidad. |
| 4 | Programar ventana de mantenimiento. |
| 5 | Realizar backup pre-actualización. |
| 6 | Aplicar parche en producción. |
| 7 | Validar funcionamiento post-actualización. |
| 8 | Documentar cambio y fecha de aplicación. |

## 5.3.5 Gestión de certificados SSL/TLS

| Acción | Descripción | Frecuencia |
| --- | --- | --- |
| Revisar fecha de vencimiento | Alertar 30-60 días antes del vencimiento | Diaria (automática) |
| Generar nueva solicitud de certificado | Crear CSR con datos actualizados | 2 meses antes vencimiento |
| Validar dominio | Completar validación requerida por CA | Según requisito CA |
| Instalar certificado | Desplegar nuevo certificado en servidor | Día de renovación |
| Validar instalación | Verificar que sitio usa nuevo certificado | Post-instalación |

## 5.3.6 Optimización de rendimiento

| Tarea | Descripción | Frecuencia |
| --- | --- | --- |
| Analizar tiempos de carga | Ejecutar prueba de rendimiento | Semanal |
| Revisar caché | Validar que caché está configurado correctamente | Semanal |
| Optimizar imágenes | Verificar compresión y resolución | Mensual |
| Limpiar logs antiguos | Archivar logs de acceso/error | Semanal |
| Analizar consultas lentas | Identificar y optimizar queries pesadas | Mensual |

## 5.3.7 Auditoría de seguridad web

| Tarea | Descripción | Frecuencia |
| --- | --- |
| Escaneo de vulnerabilidades | Ejecutar herramientas de análisis de seguridad | Mensual |
| Validación de HTTPS | Verificar que todo tráfico es cifrado | Semanal |
| Auditoría de permisos | Revisar acceso a archivos y directorios | Trimestral |
| Análisis de inyecciones | Probar contra SQL injection, XSS, etc. | Trimestral |
| Validación de política de seguridad | Revisar headers de seguridad HTTP | Mensual |

## 5.3.8 Gestión de incidencias de mantenimiento

| Incidencia | Actuación recomendada |
| --- | --- |
| Web no disponible | Verificar conectividad y estado del servidor. |
| Certificado vencido | Instalar nuevo certificado inmediatamente. |
| Rendimiento degradado | Revisar caché, carga de servidor y optimizar. |
| Contenido corrupto | Restaurar desde backup más reciente. |
| Ataque de seguridad | Activar plan de respuesta ante incidente. |

## 5.3.9 Procedimiento de backup y recuperación web

| Acción | Descripción | Frecuencia |
| --- | --- | --- |
| Backup completo | Copia de archivos y base de datos | Diaria |
| Backup incremental | Cambios desde último backup completo | Cada 4 horas |
| Validación de backup | Verificar integridad de archivo | Semanal |
| Prueba de restauración | Simular recuperación en entorno aislado | Mensual |
| Retención de backups | Mantener histórico de mínimo 30 días | Continua |

## 5.3.10 Buenas prácticas de mantenimiento

1. Mantener proceso de actualización documentado y probado.
2. Realizar cambios durante ventanas de baja demanda (noches/fines de semana).
3. Comunicar planificación de mantenimiento a usuarios con antelación.
4. Mantener monitorización 24/7 de disponibilidad.
5. Ejecutar auditoría de seguridad regularmente.
