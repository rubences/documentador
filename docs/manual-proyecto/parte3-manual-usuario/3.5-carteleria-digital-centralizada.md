# 3.5 Actuación 5: Cartelería Digital Centralizada

## 3.5.1 Objeto del manual de usuario

La presente sección describe el uso funcional del sistema de cartelería digital centralizada
por parte del personal municipal encargado de la programación de contenidos y seguimiento
operativo de paneles interiores y exteriores.

## 3.5.2 Perfiles de uso

| Perfil | Uso principal |
| --- | --- |
| Editor de contenidos | Carga y actualización de material multimedia |
| Operador de cartelería | Programación de campañas y parrillas |
| Supervisor municipal | Revisión de estado y validación de difusión |

## 3.5.3 Acceso al software de gestión

| Paso | Instrucción |
| --- | --- |
| 1 | Acceder al entorno web de gestión de cartelería digital. |
| 2 | Iniciar sesión con credenciales autorizadas por el Ayuntamiento. |
| 3 | Verificar permisos de acceso a dispositivos y canales asignados. |
| 4 | Comprobar estado general de paneles antes de publicar contenidos. |

## 3.5.4 Programación de contenidos

| Acción | Instrucción de uso |
| --- | --- |
| Crear campaña | Definir nombre, periodo de vigencia y objetivo informativo. |
| Seleccionar soportes | Asignar paneles de interior, exterior o ambos. |
| Cargar contenido | Subir imágenes, vídeos o textos desde el gestor centralizado. |
| Definir diseño de pantalla | Elegir formato completo o distribución en 2, 3 o 4 áreas. |
| Programar horario | Establecer franja horaria y calendario de emisión. |
| Publicar campaña | Confirmar y activar la distribución remota. |

## 3.5.5 Operación diaria y seguimiento

| Acción | Instrucción |
| --- | --- |
| Consultar estado de paneles | Revisar conectividad y sincronización en el panel de monitorización. |
| Verificar actualización | Confirmar recepción local del contenido en cada pantalla. |
| Revisar contenido en emisión | Validar visualización correcta por ubicación y horario. |
| Modificar campaña activa | Ajustar contenidos o horarios sin interrumpir el servicio general. |

## 3.5.6 Gestión de incidencias de usuario

| Incidencia | Actuación recomendada |
| --- | --- |
| Un panel no muestra contenido | Revisar estado de sincronización y conectividad del dispositivo. |
| Contenido desactualizado | Forzar actualización remota y verificar versión de campaña activa. |
| Diseño visual incorrecto | Revisar la configuración de layout y el formato del contenido cargado. |
| No se publica una campaña | Verificar fechas, horarios, permisos y estado de los soportes asignados. |

## 3.5.7 Buenas prácticas

1. Validar previamente los contenidos en entorno de prueba cuando exista.
2. Evitar solapes de campañas en un mismo soporte sin prioridad definida.
3. Mantener nomenclatura homogénea para facilitar búsquedas y auditoría.
4. Registrar cambios relevantes de programación para trazabilidad interna.
