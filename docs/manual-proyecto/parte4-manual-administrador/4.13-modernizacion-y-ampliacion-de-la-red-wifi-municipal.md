# 4.13 Actuación 13: Modernización y Ampliación de la Red WiFi Municipal - Administración

## 4.13.1 Objeto del manual de administración

La presente sección describe las tareas de administración técnica de la red WiFi municipal
modernizada, incluyendo gestión de puntos de acceso, configuración de políticas de red y
optimización de cobertura.

## 4.13.2 Perfiles de administración

| Perfil | Responsabilidades |
| --- | --- |
| Administrador de red WiFi | Configuración de infraestructura y puntos de acceso |
| Responsable de seguridad | Políticas de acceso, autenticación y cifrado |
| Técnico de mantenimiento | Monitorización de estado y resolución de incidencias |

## 4.13.3 Acceso al panel de administración WiFi

| Paso | Instrucción |
| --- | --- |
| 1 | Acceder al panel de gestión centralizado de red WiFi. |
| 2 | Iniciar sesión con credenciales de administrador de red. |
| 3 | Verificar acceso a configuración de puntos de acceso. |
| 4 | Confirmar disponibilidad de herramientas de monitorización. |

## 4.13.4 Gestión del inventario de puntos de acceso

| Acción | Instrucción |
| --- | --- |
| Registrar nuevo AP | Documentar ubicación, modelo, serie y fecha de instalación. |
| Configurar identificación | Asignar SSID, contraseña y parámetros de red. |
| Agrupar por zona | Crear conjuntos de AP para gestión coordinada. |
| Mantener inventario | Actualizar estado de hardware, retiradas o traslados. |

## 4.13.5 Configuración de red y conectividad

| Acción | Instrucción |
| --- | --- |
| Configurar modo 802.11ax | Habilitar estándar WiFi 6 según especificación de hardware. |
| Definir canal de operación | Asignar canales para evitar interferencia. |
| Configurar ancho de banda | Establecer 20/40/80/160 MHz según ubicación. |
| Validar cobertura | Verificar señal en área objetivo mediante medición in-situ. |

## 4.13.6 Políticas de acceso y autenticación

| Acción | Instrucción |
| --- | --- |
| Configurar tipos de red | Definir redes públicas, privadas o corporativas. |
| Establecer autenticación | Habilitar WPA3 u otros mecanismos de seguridad. |
| Configurar portal cautivo | Implementar pantalla de bienvenida si aplica. |
| Gestionar excepciones | Permitir acceso sin contraseña a visitantes si municipio lo requiere. |

## 4.13.7 Seguridad de red

| Acción | Instrucción |
| --- | --- |
| Configurar cifrado | Habilitar WPA3 o equivalente en todos los AP. |
| Cambiar credenciales default | Actualizar contraseña de administración de equipos. |
| Habilitar firma de firmware | Activar verificación de autenticidad de actualizaciones. |
| Auditar políticas de acceso | Revisar quién tiene acceso administrativo a AP. |

## 4.13.8 Monitorización de estado

| Acción | Instrucción |
| --- | --- |
| Revisar conectividad de AP | Verificar estado online de cada punto y última sincronización. |
| Monitorizar carga de tráfico | Supervisar ancho de banda utilizado y conexiones activas. |
| Revisar eventos de error | Analizar logs de desconexiones o problemas de comunicación. |
| Ejecutar diagnóstico remoto | Lanzar prueba de funcionalidad desde consola central. |

## 4.13.9 Actualización y mantenimiento

| Acción | Instrucción |
| --- | --- |
| Descargar actualización | Obtener firmware más reciente del repositorio de proveedor. |
| Programar actualización | Agendar en ventana de baja demanda para usuarios. |
| Distribuir remotamente | Aplicar firmware a múltiples AP sin intervención in-situ. |
| Validar compatibilidad | Verificar funcionamiento de servicios tras actualización. |

## 4.13.10 Análisis de cobertura y capacidad

| Acción | Instrucción |
| --- | --- |
| Medir señal en campo | Realizar site survey con herramientas de medición. |
| Identificar puntos muertos | Detectar áreas con cobertura débil o nula. |
| Planificar expansión | Proponer nuevos AP para zonas no cubiertas. |
| Analizar rendimiento | Evaluar velocidad efectiva y consistencia de conexión. |

## 4.13.11 Gestión de incidencias administrativas

| Incidencia | Actuación recomendada |
| --- | --- |
| AP offline prolongado | Revisar conectividad eléctrica y de red del equipo. |
| Cobertura débil en zona | Ejecutar site survey y considerar reubicación o nuevo AP. |
| Usuarios no pueden conectar | Verificar SSID visible, autenticación y compatibilidad de dispositivo. |
| Rendimiento bajo | Monitorizar interferencias, carga de tráfico y optimizar canales. |

## 4.13.12 Buenas prácticas administrativas

1. Realizar inventario completo de AP semestralmente.
2. Ejecutar site survey anual para validar cobertura.
3. Documenta cambios de configuración y auditar regularmente.
4. Mantener log de actualizaciones de firmware y compatibilidades.
5. Revisar periódicamente estadísticas de uso para planificación de capacidad.
