# 3.6 Actuación 6: Señalización Turística Urbana Inteligente

## 3.6.1 Objeto del manual de usuario

La presente sección describe el uso funcional de la señalización turística inteligente por parte
de personal municipal y usuarios finales en interacción con QR, NFC y beacons.

## 3.6.2 Uso por visitante

| Acción de usuario | Resultado esperado |
| --- | --- |
| Escanear QR | Acceso directo a mapas, rutas y recursos turísticos |
| Acercar móvil a NFC | Apertura de información contextual sin contacto |
| Entrar en rango BLE | Recepción de notificaciones turísticas configuradas |
| Acceder a itinerarios | Consulta de recursos y contenidos vinculados a la señal |

## 3.6.3 Uso por operador municipal

| Acción | Instrucción de uso |
| --- | --- |
| Acceder a plataforma de beacons | Iniciar sesión en la consola online de gestión. |
| Configurar un beacon | Definir identificador, rango y contenido asociado. |
| Programar mensajes | Publicar notificaciones por perfil o por evento. |
| Actualizar destino de QR dinámico | Cambiar contenido sin intervención física sobre la señal. |
| Verificar funcionamiento | Ejecutar prueba en dispositivo móvil en campo. |

## 3.6.4 Flujo operativo recomendado

| Paso | Instrucción |
| --- | --- |
| 1 | Revisar inventario de señales/tótems activos. |
| 2 | Confirmar contenido vigente por punto de interés. |
| 3 | Programar o actualizar mensajes contextuales. |
| 4 | Validar lectura QR/NFC y recepción BLE en pruebas reales. |
| 5 | Registrar cambios en la bitácora operativa del servicio. |

## 3.6.5 Gestión de incidencias de usuario

| Incidencia | Actuación recomendada |
| --- | --- |
| QR no abre contenido | Verificar URL asociada y actualizar enlace en plataforma. |
| NFC no responde | Revisar compatibilidad del terminal y estado del chip en señal. |
| No llega notificación BLE | Comprobar Bluetooth, rango efectivo y configuración del beacon. |
| Contenido desactualizado | Revisar programación y aplicar actualización remota. |

## 3.6.6 Buenas prácticas

1. Mantener coherencia entre contenido del CMS y contenido vinculado a señales.
2. Revisar periódicamente rutas de alto tránsito turístico.
3. Configurar mensajes breves y accionables para usuarios en movilidad.
4. Verificar accesibilidad del contenido cuando se habiliten soluciones específicas.
