# 5.2 Actuación 2: Gestor de Contenidos Turísticos (CMS) - Mantenimiento

## 5.2.1 Objeto del manual de mantenimiento

La presente sección describe los procedimientos de mantenimiento preventivo y correctivo del
Gestor de Contenidos Turísticos, incluyendo limpieza de datos, optimización de base de datos
y auditoría de integridad de contenidos.

## 5.2.2 Equipamiento y componentes sujetos a mantenimiento

| Componente | Tipo de mantenimiento | Frecuencia |
| --- | --- | --- |
| Base de datos de CMS | Optimización, integridad, backup | Semanal |
| Motor de búsqueda | Reindexación de contenidos | Quincenal |
| Caché de contenido | Limpieza y renovación | Semanal |
| Flujos de aprobación | Auditoría de estados pendientes | Diaria |
| Almacenamiento de archivos | Validación de integridad de multimedia | Mensual |

## 5.2.3 Plan de mantenimiento preventivo

| Tarea | Descripción | Frecuencia | Responsable |
| --- | --- | --- | --- |
| Validar integridad de contenido | Verificar que contenido publicado es accesible | Diaria | Script automatizado |
| Limpiar borradores obsoletos | Eliminar borradores sin actualizar hace >6 meses | Mensual | Administrador CMS |
| Reindexar búsqueda | Actualizar índices del motor de búsqueda | Quincenal | Automatizado |
| Auditar taxonomía | Validar que etiquetas y categorías son coherentes | Mensual | Responsable taxonomía |
| Backup de contenido | Realizar copia de seguridad de base de datos | Diaria | Automatizado |

## 5.2.4 Procedimiento de limpieza de contenido

| Paso | Instrucción |
| --- | --- |
| 1 | Identificar contenido obsoleto según política (edad, estado). |
| 2 | Auditar referencias antes de eliminar. |
| 3 | Crear backup del contenido a eliminar. |
| 4 | Ejecutar eliminación en base de datos. |
| 5 | Validar que no hay referencias rotas post-eliminación. |
| 6 | Documentar contenido eliminado y razón. |

## 5.2.5 Optimización de base de datos de CMS

| Tarea | Descripción | Frecuencia |
| --- | --- | --- |
| Reorganización de índices | Reconstruir índices fragmentados | Mensual |
| Análisis de estadísticas | Actualizar estadísticas de tablas | Semanal |
| Eliminación de registros huérfanos | Limpiar registros sin relación válida | Mensual |
| Validación de integridad | Ejecutar chequeo de restricciones | Trimestral |

## 5.2.6 Gestión de multimedia y almacenamiento

| Acción | Descripción | Frecuencia |
| --- | --- | --- |
| Validación de archivos | Verificar que multimedia está accesible | Mensual |
| Limpieza de archivos huérfanos | Eliminar multimedia sin referencias | Trimestral |
| Optimización de imágenes | Comprobar que están adecuadamente comprimidas | Trimestral |
| Auditoría de permisos | Validar que acceso a archivos es correcto | Semestral |

## 5.2.7 Auditoría de flujos editoriales

| Tarea | Descripción | Frecuencia |
| --- | --- | --- |
| Revisar borradores atrapados | Identificar contenido en estado pendiente >2 semanas | Semanal |
| Auditar cambios sin aprobación | Validar que solo contenido aprobado está publicado | Diaria |
| Revisar permisos de editor | Verificar que permisos asignados son correctos | Mensual |
| Análisis de auditoría | Generar informe de cambios por editor | Trimestral |

## 5.2.8 Procedimiento de recuperación ante corrupción

| Paso | Instrucción |
| --- | --- |
| 1 | Detectar corrupción mediante validación de integridad. |
| 2 | Aislar sistema y notificar a usuarios. |
| 3 | Restaurar desde backup más reciente. |
| 4 | Validar integridad de datos restaurados. |
| 5 | Sincronizar cambios post-backup si es posible. |
| 6 | Documentar incidente y causa de corrupción. |

## 5.2.9 Gestión de incidencias de mantenimiento

| Incidencia | Actuación recomendada |
| --- | --- |
| Búsqueda no encuentra contenido | Ejecutar reindexación completa del motor. |
| Contenido desaparece sin razón | Restaurar desde backup y investigar causa. |
| Rendimiento degradado | Ejecutar análisis de índices y optimizar tablas. |
| Flujo de aprobación atascado | Revisar reglas y liberar contenido manualmente si aplica. |

## 5.2.10 Buenas prácticas de mantenimiento

1. Realizar limpieza de contenido según política documentada.
2. Mantener backup diario con pruebas de restauración mensuales.
3. Monitorizar crecimiento de base de datos para planificación de capacidad.
4. Documentar todas las limpiezas y auditorías realizadas.
5. Coordinar limpieza de contenido con responsables editoriales.
