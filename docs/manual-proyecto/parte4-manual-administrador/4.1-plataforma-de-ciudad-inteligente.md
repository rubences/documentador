# 4.1 Actuación 1: Plataforma de Ciudad Inteligente (PCI) - Administración

## 4.1.1 Objeto del manual de administración

La presente sección describe las tareas de administración, configuración y gestión avanzada
de la Plataforma de Ciudad Inteligente destinadas al personal técnico administrativo del
Ayuntamiento.

## 4.1.2 Perfiles de administración

| Perfil | Responsabilidades |
| --- | --- |
| Administrador de sistemas | Configuración de infraestructura, usuarios y roles |
| Administrador de aplicación | Gestión de módulos, integraciones y actualizaciones |
| Responsable de seguridad | Auditoría, políticas de acceso y criptografía |

## 4.1.3 Acceso al panel administrativo

| Paso | Instrucción |
| --- | --- |
| 1 | Acceder a la URL de administración de la Plataforma. |
| 2 | Autenticarse con credenciales de administrador de sistemas. |
| 3 | Validar segundo factor de autenticación si está configurado. |
| 4 | Confirmar acceso al módulo administrativo con todas las opciones habilitadas. |

## 4.1.4 Gestión de usuarios y roles

| Acción | Instrucción |
| --- | --- |
| Crear usuario | Definir nombre, correo, rol inicial y unidad organizativa. |
| Asignar roles | Otorgar permisos según perfil (consulta, edición, gestión de alertas). |
| Modificar permisos | Ajustar acceso a módulos y funcionalidades específicas. |
| Desactivar usuario | Revocar acceso manteniendo histórico de acciones realizadas. |
| Resetear contraseña | Regenerar credenciales mediante procedimiento seguro. |

## 4.1.5 Configuración de integración de datos

| Acción | Instrucción |
| --- | --- |
| Configurar origen de datos | Añadir conexión a fuente externa (API, BBDD, fichero). |
| Validar conectividad | Ejecutar prueba de comunicación con cada origen. |
| Definir frecuencia de sincronización | Establecer intervalo de actualización de datos. |
| Monitorizar estado de integración | Revisar eventos y errores de sincronización reciente. |

## 4.1.6 Auditoría y trazabilidad

| Acción | Instrucción |
| --- | --- |
| Consultar registro de acceso | Revisar quién accedió, cuándo y desde qué origen. |
| Auditar cambios de configuración | Identificar modificaciones a roles, permisos o integraciones. |
| Revisar eventos de sistema | Consultar alertas técnicas y cambios en el estado de servicios. |
| Exportar informe de auditoría | Descargar registro para análisis de compliance. |

## 4.1.7 Actualización y mantenimiento

| Acción | Instrucción |
| --- | --- |
| Consultar versión actual | Verificar build de aplicación en interfaz administrativa. |
| Planificar actualización | Agendar actualización en ventana de baja demanda. |
| Ejecutar actualización | Seguir procedimiento de upgrade sin afectar datos existentes. |
| Validar compatibilidad | Revisar integraciones tras cambio de versión. |

## 4.1.8 Gestión de incidencias administrativas

| Incidencia | Actuación recomendada |
| --- | --- |
| Usuario sin acceso | Verificar estado de cuenta, roles y políticas de acceso. |
| Integración fallida | Revisar conectividad de origen de datos y credenciales. |
| Rendimiento degradado | Monitorizar carga de sistema y optimizar consultas. |
| Inconsistencia de datos | Validar integridad y ejecutar reconciliación si procede. |

## 4.1.9 Buenas prácticas administrativas

1. Revisar regularmente cambios en el registro de auditoría.
2. Mantener principio de menor privilegio en asignación de roles.
3. Documentar todos los cambios de configuración crítica.
4. Ejecutar backups automáticos al menos diariamente.
5. Revisar periódicamente credenciales y políticas de contraseña.
