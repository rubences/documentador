# 4.10 Actuación 10: Sistema de Control Inteligente del Tráfico - Administración

## 4.10.1 Objeto del manual de administración

La presente sección describe las tareas de administración técnica del sistema inteligente de
gestión semafórica, incluyendo configuración de controladores, parametrización de ciclos y
auditoría de cambios operativos.

## 4.10.2 Perfiles de administración

| Perfil | Responsabilidades |
| --- | --- |
| Administrador de tráfico | Configuración de intersecciones y planes de tráfico |
| Responsable de timing | Análisis y ajuste de ciclos semafóricos |
| Técnico de comunicaciones | Mantenimiento de conectividad de controladores |

## 4.10.3 Acceso al módulo administrativo de tráfico

| Paso | Instrucción |
| --- | --- |
| 1 | Acceder a la consola de administración del sistema semafórico. |
| 2 | Iniciar sesión con credenciales de administrador de tráfico. |
| 3 | Verificar acceso a configuración de intersecciones y planes. |
| 4 | Confirmar disponibilidad de herramientas de análisis de flujo. |

## 4.10.4 Gestión de intersecciones y controladores

| Acción | Instrucción |
| --- | --- |
| Registrar nueva intersección | Documentar ubicación, identificador y controladores vinculados. |
| Configur ar tipo de controlador | Seleccionar modelo y versión de equipo que gestiona la intersección. |
| Establecer comunicación | Validar conectividad entre controlador y sistema central. |
| Asignar permisos de cambio | Definir qué usuarios pueden modificar parámetros de cada intersección. |

## 4.10.5 Configuración de planes de tráfico

| Acción | Instrucción |
| --- | --- |
| Crear plan de tráfico | Definir nuevo conjunto de ciclos y tiempos. |
| Configurar fases | Establecer duración de verde, ámbar y rojo por fase. |
| Optimizar ciclo | Ajustar tempo basado en análisis de flujos históricos. |
| Versionar plan | Mantener histórico de cambios para rollback si es necesario. |

## 4.10.6 Aplicación y cambio de planes

| Acción | Instrucción |
| --- | --- |
| Programar cambio de plan | Agendar aplicación en ventana de bajo impacto. |
| Aplicar plan en tiempo real | Cambiar configuración operativa de intersección. |
| Validar efectividad | Monitorizar flujo tras cambio para verificar mejora. |
| Revertir cambio | Restaurar plan anterior si cambio degrada tráfico. |

## 4.10.7 Monitorización técnica de controladores

| Acción | Instrucción |
| --- | --- |
| Revisar estado de equipos | Verificar conectividad y estado operativo de cada controlador. |
| Auditar eventos de error | Revisar alertas y códigos de fallo reportados. |
| Revisar logs de actuación | Analizar historial de cambios de semáforo y eventos. |
| Ejecutar diagnóstico remoto | Lanzar prueba de comunicación y respuesta del equipo. |

## 4.10.8 Análisis de datos de tráfico

| Acción | Instrucción |
| --- | --- |
| Generar informe de flujo | Crear resumen de vehículos y congestión por período. |
| Analizar tendencias | Identificar horas punta y patrones de tráfico. |
| Comparar performance | Evaluar efectividad de planes en diferentes intersecciones. |
| Exportar para análisis | Proporcionar datos a sistemas de planificación de movilidad. |

## 4.10.9 Auditoría y conformidad

| Acción | Instrucción |
| --- | --- |
| Revisar registro de cambios | Acceder a log de toda modificación de configuración. |
| Auditar permisos | Verificar que cambios fueron realizados por usuario autorizado. |
| Generar informe de cumplimiento | Documentar adherencia a procedimientos y estándares. |
| Gestionar cambios excepcionales | Registrar cambios de emergencia con justificación. |

## 4.10.10 Gestión de incidencias administrativas

| Incidencia | Actuación recomendada |
| --- | --- |
| Intersección sin comunicación | Revisar conectividad de red y estado del controlador. |
| Plan no se aplica correctamente | Verificar sintaxis de configuración y versión del equipo. |
| Flujo congestionado tras cambio | Revertir a plan anterior y análizar causa de degradación. |
| Alerta recurrente | Investigar causa técnica y aplicar corrección en hardware o firmware. |

## 4.10.11 Buenas prácticas administrativas

1. Documentar razón y responsable de cada cambio en plan de tráfico.
2. Realizar cambios de configuración en horas de baja demanda cuando sea posible.
3. Probar nuevos planes en simulación antes de aplicación en campo.
4. Revisar mensualmente alertas técnicas de controladores.
5. Mantener comunicación con Policía Local sobre cambios importantes.
