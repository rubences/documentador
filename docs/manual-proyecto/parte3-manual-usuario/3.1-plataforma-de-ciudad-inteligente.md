# 3.1 Actuación 1: Plataforma de Ciudad Inteligente (PCI)

## 3.1.1 Objeto del manual de usuario

La presente sección describe las instrucciones básicas de uso de la Plataforma de Ciudad
Inteligente para perfiles usuarios de consulta y explotación funcional, sin entrar en tareas de
administración avanzada o mantenimiento.

Conforme al Pliego, este manual forma parte del compendio de instrucciones de usuario de los
elementos del sistema y debe facilitar una utilización autónoma por parte del personal
municipal habilitado.

## 3.1.2 Perfiles de usuario

| Perfil | Uso principal |
| --- | --- |
| Usuario de consulta | Visualización de datos, indicadores y cuadros de mando |
| Usuario gestor funcional | Revisión de eventos, alertas y explotación operativa de la información |
| Usuario analista | Consulta de información histórica y análisis comparativo |

## 3.1.3 Acceso a la plataforma

| Paso | Instrucción de uso |
| --- | --- |
| 1 | Acceder a la URL corporativa o entorno web de la Plataforma facilitado por el Ayuntamiento. |
| 2 | Introducir las credenciales de acceso asignadas por el área técnica municipal. |
| 3 | En caso de segundo factor o certificado, completar el proceso de autenticación definido en la implantación. |
| 4 | Verificar que se visualiza el panel principal con los módulos autorizados para el perfil del usuario. |

## 3.1.4 Uso básico del panel principal

| Acción | Descripción |
| --- | --- |
| Consultar indicadores | Permite visualizar cuadros de mando, métricas e información consolidada del destino. |
| Filtrar información | Permite acotar consultas por fecha, zona, dispositivo, tipo de evento o servicio. |
| Revisar información en tiempo real | Permite consultar estado actual de fuentes integradas y eventos activos. |
| Revisar histórico | Permite acceder a series históricas y comparativas temporales. |
| Exportar resultados | Permite descargar la información habilitada por el sistema para su análisis posterior. |

## 3.1.5 Gestión de alertas y eventos

| Acción | Instrucción |
| --- | --- |
| Consultar alertas activas | Acceder al módulo de alertas o avisos desde el panel principal. |
| Revisar detalle de una alerta | Seleccionar la alerta y consultar origen, fecha, severidad y elemento afectado. |
| Confirmar revisión | Registrar, si el sistema lo permite, la validación o revisión funcional del aviso. |
| Escalar incidencia | Comunicar al perfil administrador o de mantenimiento cuando la alerta requiera actuación técnica. |

## 3.1.6 Consulta de información geográfica

| Acción | Instrucción |
| --- | --- |
| Abrir visor geográfico | Acceder al módulo cartográfico o GIS de la Plataforma. |
| Localizar activos y eventos | Visualizar sensores, dispositivos o incidencias sobre el mapa. |
| Consultar detalle georreferenciado | Seleccionar el elemento sobre el mapa para desplegar su ficha de información. |

## 3.1.7 Buenas prácticas de uso

1. Utilizar exclusivamente las credenciales personales asignadas.
2. No exportar información fuera de los canales autorizados por el Ayuntamiento.
3. Revisar diariamente alertas y eventos si el perfil incluye seguimiento operativo.
4. Cerrar sesión al finalizar el uso, especialmente en equipos compartidos.

## 3.1.8 Incidencias frecuentes de usuario

| Incidencia | Actuación recomendada |
| --- | --- |
| No se puede iniciar sesión | Verificar credenciales y contactar con soporte si persiste. |
| No se visualizan módulos esperados | Comprobar el perfil asignado y solicitar revisión de permisos. |
| Un cuadro de mando no carga datos | Actualizar la vista y comunicar incidencia si el problema persiste. |
| No aparece una fuente o dispositivo | Comprobar filtros activos y, en su caso, trasladar la incidencia al administrador. |
