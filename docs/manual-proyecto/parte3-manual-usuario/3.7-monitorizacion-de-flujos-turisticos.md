# 3.7 Actuación 7: Monitorización de Flujos Turísticos

## 3.7.1 Objeto del manual de usuario

La presente sección describe el uso funcional del sistema de monitorización de flujos por
personal municipal encargado de supervisión operativa, consulta de indicadores y gestión de
alertas de aforo.

## 3.7.2 Perfiles de uso

| Perfil | Uso principal |
| --- | --- |
| Operador de movilidad | Seguimiento en tiempo real de aforos y flujos |
| Analista municipal | Consulta histórica y generación de informes |
| Responsable de seguridad | Atención a alertas por umbrales críticos |

## 3.7.3 Acceso al dashboard

| Paso | Instrucción |
| --- | --- |
| 1 | Acceder al panel web o aplicación habilitada para el sistema de aforos. |
| 2 | Iniciar sesión con usuario autorizado. |
| 3 | Seleccionar zona, periodo y modo de visualización (tiempo real o histórico). |
| 4 | Confirmar conectividad de nodos antes de iniciar la supervisión diaria. |

## 3.7.4 Operación en tiempo real

| Acción | Instrucción |
| --- | --- |
| Monitorizar entradas y salidas | Revisar el conteo bidireccional por zona monitorizada. |
| Revisar ocupación instantánea | Consultar el aforo actual en cada punto. |
| Atender alertas de saturación | Verificar umbral alcanzado y activar protocolo municipal. |
| Visualizar mapas de calor | Identificar áreas de alta concentración para decisiones operativas. |

## 3.7.5 Consulta histórica y reporting

| Acción | Instrucción |
| --- | --- |
| Seleccionar intervalo temporal | Elegir día, semana, mes u otro periodo configurado. |
| Comparar zonas | Analizar variación entre ubicaciones o periodos equivalentes. |
| Exportar informe | Descargar estadísticas para análisis turístico y planificación. |
| Registrar observaciones | Añadir notas de contexto para seguimiento interno. |

## 3.7.6 Gestión de incidencias de usuario

| Incidencia | Actuación recomendada |
| --- | --- |
| No aparecen datos de un punto | Revisar estado del nodo y filtros aplicados en el panel. |
| Conteos incoherentes | Comprobar configuración de zona de conteo y registrar revisión. |
| No se genera alerta esperada | Verificar umbral y configuración de notificaciones. |
| No se exporta informe | Revisar permisos del perfil y rango de datos seleccionado. |

## 3.7.7 Buenas prácticas

1. Verificar al inicio de jornada la disponibilidad de todos los nodos monitorizados.
2. Usar criterios homogéneos de periodos para comparativas históricas.
3. Documentar incidencias y acciones realizadas ante alertas críticas.
4. Mantener coordinación con perfiles de seguridad y movilidad cuando haya eventos masivos.
