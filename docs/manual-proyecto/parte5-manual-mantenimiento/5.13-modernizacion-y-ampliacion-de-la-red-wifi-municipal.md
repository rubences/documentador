# 5.13 Actuación 13: Modernización y Ampliación de la Red WiFi Municipal - Mantenimiento

## 5.13.1 Objeto del manual de mantenimiento

La presente sección describe los procedimientos de mantenimiento preventivo y correctivo de
la red WiFi municipal modernizada, incluyendo actualización de firmware de AP, validación
de cobertura y gestión de fallos de equipamiento.

## 5.13.2 Equipamiento y componentes sujetos a mantenimiento

| Componente | Tipo de mantenimiento | Frecuencia |
| --- | --- | --- |
| Puntos de acceso WiFi | Firmware, monitorización de fallos | Trimestral |
| Antenas y conectores | Inspección visual, limpieza | Trimestral |
| Sistemas de energía | Validación de suministro y baterías | Mensual |
| Conectividad de backhaul | Validación de enlaces de comunicación | Semanal |
| Infraestructura de central | Monitorización de rendimiento | Diaria |

## 5.13.3 Plan de mantenimiento preventivo

| Tarea | Descripción | Frecuencia | Responsable |
| --- | --- | --- | --- |
| Monitorización de AP | Verificar que todos los AP están online | Continua (automática) | Sistema |
| Test de cobertura | Validar señal en área de cobertura esperada | Trimestral | Especialista RF |
| Auditoría de seguridad | Revisar configuración de wifi 6 y cifrado | Mensual | Equipo seguridad |
| Actualización de firmware | Instalar versión más reciente | Trimestral | Administrador |
| Auditoría de rendimiento | Revisar velocidad y latencia | Semanal | Equipo técnico |

## 5.13.4 Procedimiento de actualización de firmware de AP

| Paso | Instrucción |
| --- | --- |
| 1 | Descargar nueva versión de firmware. |
| 2 | Revisar notas de release para cambios relevantes. |
| 3 | Planificar actualización en ventana de bajo uso. |
| 4 | Realizar backup de configuración de AP. |
| 5 | Ejecutar actualización desde consola central. |
| 6 | Validar que AP está online post-actualización. |
| 7 | Verificar que servicios WiFi están operativos. |
| 8 | Documentar actualización realizada. |

## 5.13.5 Validación de cobertura

| Tarea | Descripción | Frecuencia |
| --- | --- | --- |
| Medición de RSSI | Revisar potencia de señal en puntos de interés | Trimestral |
| Identificación de dead zones | Detectar áreas sin cobertura suficiente | Trimestral |
| Test de velocidad | Validar throughput en diferentes ubicaciones | Trimestral |
| Análisis de interferencias | Detectar canales con congestión | Mensual |

## 5.13.6 Procedimiento de site survey

| Paso | Instrucción |
| --- | --- |
| 1 | Planificar puntos de medición en área de cobertura. |
| 2 | Utilizar herramienta de análisis de WiFi. |
| 3 | Medir RSSI en cada punto. |
| 4 | Documentar ubicación y valores obtenidos. |
| 5 | Identificar áreas con cobertura débil (<-70dBm). |
| 6 | Proponer ajustes o instalación de nuevos AP si necesario. |
| 7 | Crear mapa de cobertura actualizado. |

## 5.13.7 Mantenimiento de antenas y conectores

| Acción | Descripción | Frecuencia |
| --- | --- | --- |
| Inspección visual | Revisar estado de antenas y conectores | Trimestral |
| Limpieza de antenas | Remover polvo y contaminación | Trimestral |
| Validación de alineación | Verificar que antenas están correctamente orientadas | Trimestral |
| Revisión de cables | Comprobar que cables no están dañados | Trimestral |

## 5.13.8 Gestión de fallos de equipamiento

| Tipo de falla | Diagnóstico | Acción correctiva |
| --- | --- | --- |
| AP offline | Revisar conectividad de red y energía | Reiniciar o reparar conexión |
| Cobertura débil en zona | Ejecutar site survey | Reposicionar AP o instalar nuevo |
| Rendimiento bajo | Auditar uso de canal y tráfico | Cambiar canal o escalar AP |
| Fallos intermitentes | Test de conectividad de backhaul | Revisar enlaces o mejorar redundancia |

## 5.13.9 Monitorización de rendimiento

| Métrica | Descripción | Frecuencia |
| --- | --- | --- |
| Disponibilidad de AP | Porcentaje de tiempo online | Continua |
| Usuarios conectados | Número de dispositivos por AP | Continua |
| Tráfico de datos | Throughput y ancho de banda utilizado | Continua |
| Calidad de servicio | Latencia y packet loss | Semanal |

## 5.13.10 Documentación de mantenimiento

| Documento | Contenido | Actualización |
| --- | --- | --- |
| Log de actualizaciones | Versión de firmware por AP, fecha, cambios | Cada actualización |
| Mapa de cobertura | RSSI, canales utilizados, dead zones | Trimestral |
| Registro de fallos | Incidencias reportadas y tiempo de resolución | Continua |
| Plan de expansión | AP propuestos para mejorar cobertura | Trimestral |

## 5.13.11 Buenas prácticas de mantenimiento

1. Ejecutar site survey anual para validar cobertura.
2. Mantener programa de actualización de firmware documentado.
3. Documentar todas las intervenciones técnicas en AP.
4. Monitorizar continuamente disponibilidad y rendimiento.
5. Planificar expansión basada en análisis de demanda y cobertura.
