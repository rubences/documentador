# 5.10 Actuación 10: Sistema de Control Inteligente del Tráfico - Mantenimiento

## 5.10.1 Objeto del manual de mantenimiento

La presente sección describe los procedimientos de mantenimiento preventivo y correctivo del
sistema inteligente de gestión semafórica, incluyendo actualización de firmware de
controladores, validación de enlaces de comunicación y gestión de fallos de equipamiento.

## 5.10.2 Equipamiento y componentes sujetos a mantenimiento

| Componente | Tipo de mantenimiento | Frecuencia |
| --- | --- | --- |
| Controladores semafóricos | Firmware, monitorización de fallos | Trimestral |
| Conexiones de red | Validación de enlaces, diagnostico | Semanal |
| Fuentes de alimentación | Inspección, test de carga | Trimestral |
| Elementos luminosos | Validación de LED, recambio de bombillas | Mensual |
| Sistema de respaldo | Validación de baterías y UPS | Mensual |

## 5.10.3 Plan de mantenimiento preventivo

| Tarea | Descripción | Frecuencia | Responsable |
| --- | --- | --- | --- |
| Validación de comunicación | Verificar que controladores responden a central | Semanal | Equipo técnico |
| Test de semáforos | Revisar que ciclo de colores es correcto | Mensual | Técnico en campo |
| Inspección visual | Revisar estado físico de cabezales y cajas | Trimestral | Equipo mantenimiento |
| Actualización de firmware | Instalar versión más reciente de software | Trimestral | Administrador |
| Auditoría de eventos | Revisar logs de incidencias y errores | Semanal | Equipo técnico |

## 5.10.4 Procedimiento de actualización de firmware

| Paso | Instrucción |
| --- | --- |
| 1 | Descargar nueva versión de firmware. |
| 2 | Revisar notas de release para impacto en operación. |
| 3 | Realizar backup de configuración actual. |
| 4 | Programar ventana de actualización en horario de bajo tráfico. |
| 5 | Ejecutar actualización desde consola central. |
| 6 | Validar que controlador responde correctamente post-actualización. |
| 7 | Verificar que ciclos semafóricos están operativos. |
| 8 | Documentar actualización realizada. |

## 5.10.5 Mantenimiento de elementos luminosos

| Acción | Descripción | Frecuencia |
| --- | --- | --- |
| Inspección visual de LED | Revisar que todos los LED encienden | Mensual |
| Test de visibilidad | Validar desde distancia que luz es visible | Mensual |
| Limpieza de óptica | Remover polvo de lentes de semáforo | Trimestral |
| Reemplazo de LED defectuoso | Sustituir LED que no funciona | As-needed |

## 5.10.6 Procedimiento de reemplazo de LED

| Paso | Instrucción |
| --- | --- |
| 1 | Identificar LED defectuoso mediante inspección. |
| 2 | Programar reemplazo en ventana de baja demanda. |
| 3 | Desactivar semáforo si es necesario para seguridad. |
| 4 | Acceder a cabezal siguiendo procedimiento de seguridad. |
| 5 | Extraer LED defectuoso. |
| 6 | Instalar LED nuevo del tipo correcto. |
| 7 | Validar que LED funciona correctamente. |
| 8 | Reactivar semáforo. |
| 9 | Documentar reemplazo realizado. |

## 5.10.7 Validación de comunicación y conectividad

| Tarea | Descripción | Frecuencia |
| --- | --- | --- |
| Test de conectividad | Ping a controlador desde consola central | Diaria (automática) |
| Validación de latencia | Verificar tiempo de respuesta es aceptable | Semanal |
| Inspección de cables | Revisar estado de conexiones de red | Trimestral |
| Test de redundancia | Validar failover si conexión primaria falla | Trimestral |

## 5.10.8 Gestión de fallos de equipamiento

| Tipo de falla | Diagnóstico | Acción correctiva |
| --- | --- | --- |
| Controlador offline | Revisar conectividad de red y energía | Reiniciar o reparar conexión |
| LED no enciende | Inspeccionar LED y conexiones | Reemplazar LED o revisar fuente |
| Ciclo no respeta timing | Recalibrar controlador o actualizar firmware | Recalibrar o actualizar |
| Comunicación intermitente | Test de enlace de comunicación | Revisar cables o reparar conexión |

## 5.10.9 Monitorización de rendimiento

| Métrica | Descripción | Frecuencia |
| --- | --- | --- |
| Uptime de controladores | Porcentaje de tiempo que controlador está online | Continua |
| Eventos de error | Número de eventos de error por controlador | Semanal |
| Tiempo de respuesta | Latencia de comunicación entre controlador y central | Semanal |
| Ciclos interrumpidos | Número de ciclos que no completaron | Mensual |

## 5.10.10 Documentación de mantenimiento

| Documento | Contenido | Actualización |
| --- | --- | --- |
| Log de actualizaciones | Versión, fecha, cambios realizados, responsable | Cada actualización |
| Registro de reemplazos | LED/componentes reemplazados, fecha, razón | Cada reemplazo |
| Historial de fallos | Incidencias reportadas y tiempo de resolución | Continua |
| Matriz de estado | Estado actual de cada intersección | Semanal |

## 5.10.11 Buenas prácticas de mantenimiento

1. Mantener programa regular de inspecciones visuales en campo.
2. Documentar todas las intervenciones técnicas realizadas.
3. Realizar actualizaciones de firmware en ventanas coordinadas.
4. Mantener stock de LED y componentes de reemplazo.
5. Ejecutar auditoría trimestral de rendimiento global del sistema.
