# 3.2 Actuación 2: Gestor de Contenidos Turísticos

## 3.2.1 Objeto del manual de usuario

La presente sección describe el uso funcional del Gestor de Contenidos Turísticos por parte de
usuarios editores y gestores de contenido municipal.

## 3.2.2 Perfiles de usuario

| Perfil | Uso principal |
| --- | --- |
| Editor de contenidos | Alta, edición y publicación de información turística |
| Gestor de rutas | Gestión de rutas, recursos geolocalizados y contenidos asociados |
| Responsable de canal | Programación y distribución de contenidos a soportes |

## 3.2.3 Acceso al gestor

| Paso | Instrucción |
| --- | --- |
| 1 | Acceder al entorno web del gestor de contenidos. |
| 2 | Introducir usuario y contraseña. |
| 3 | Completar, si procede, el segundo factor de autenticación configurado. |
| 4 | Comprobar que se muestran los menús correspondientes al rol asignado. |

## 3.2.4 Gestión de contenidos informativos

| Acción | Instrucción de uso |
| --- | --- |
| Crear contenido | Seleccionar la tipología correspondiente y completar los campos obligatorios. |
| Editar contenido | Acceder al listado, localizar el contenido y abrir el editor WYSIWYG o formulario habilitado. |
| Publicar contenido | Guardar y lanzar la publicación conforme al flujo definido en la implantación. |
| Despublicar contenido | Retirar el contenido desde la gestión del estado de publicación. |
| Adjuntar archivos | Cargar imágenes, vídeos, audios, PDF u otros ficheros auxiliares desde la biblioteca. |

## 3.2.5 Gestión de categorías, rutas y recursos

| Acción | Instrucción |
| --- | --- |
| Crear categoría | Acceder al módulo de taxonomías o categorías y dar de alta una nueva entrada. |
| Organizar jerarquía | Reordenar categorías y subcategorías según la estructura de navegación definida. |
| Crear ruta turística | Dar de alta la ruta y asociar hitos, coordenadas y recursos vinculados. |
| Asociar contenidos a un recurso | Añadir descripciones, multimedia, QR, accesibilidad y relaciones cercanas. |

## 3.2.6 Distribución multicanal de contenidos

| Acción | Instrucción |
| --- | --- |
| Programar parrillas | Seleccionar soportes, franjas horarias y fechas de vigencia. |
| Asignar contenido por ubicación | Vincular contenidos a soportes o grupos de soportes según localización. |
| Lanzar emisión a demanda | Publicar un contenido concreto sin esperar a una programación previa. |
| Probar una parrilla | Ejecutar la previsualización o prueba antes de la distribución definitiva. |

## 3.2.7 Gestión de usuarios y roles

| Acción | Instrucción |
| --- | --- |
| Crear usuario | Acceder al módulo de usuarios y completar los datos requeridos. |
| Asignar rol | Seleccionar el rol funcional correspondiente al usuario. |
| Modificar permisos | Ajustar privilegios de acceso y edición según perfil. |
| Desactivar usuario | Bloquear o retirar acceso cuando deje de ser necesario. |

## 3.2.8 Buenas prácticas de uso

1. Validar la calidad y actualidad del contenido antes de publicarlo.
2. Mantener nomenclaturas homogéneas en categorías, recursos y rutas.
3. Utilizar siempre los campos de idioma cuando el contenido deba mostrarse en varios idiomas.
4. Verificar la programación antes de difundir contenido a cartelería, web o app.

## 3.2.9 Incidencias frecuentes de usuario

| Incidencia | Actuación recomendada |
| --- | --- |
| No aparece un contenido publicado | Verificar estado de publicación, idioma y programación asignada. |
| Un archivo no se visualiza correctamente | Revisar formato, carga y asociación con el contenido. |
| No se puede editar un recurso | Comprobar permisos de rol o bloqueo del contenido. |
| La distribución a soportes no se refleja | Revisar parrilla, fechas y canal asignado antes de escalar incidencia. |
