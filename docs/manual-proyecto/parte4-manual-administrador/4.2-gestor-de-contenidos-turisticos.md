# 4.2 Actuación 2: Gestor de Contenidos Turísticos (CMS) - Administración

## 4.2.1 Objeto del manual de administración

La presente sección describe las tareas de administración técnica del Gestor de Contenidos
Turísticos, incluyendo gestión de usuarios, taxonomía, flujos de aprobación y auditoría de
cambios de contenido.

## 4.2.2 Perfiles de administración

| Perfil | Responsabilidades |
| --- | --- |
| Administrador del CMS | Configuración de plantillas, flujos y acceso a contenidos |
| Responsable de taxonomía | Gestión de categorías, etiquetas y metadatos |
| Auditor de contenidos | Revisión de cambios y cumplimiento de normativas |

## 4.2.3 Acceso al módulo administrativo del CMS

| Paso | Instrucción |
| --- | --- |
| 1 | Acceder a la zona administrativa del Gestor de Contenidos. |
| 2 | Iniciar sesión con credenciales de administrador. |
| 3 | Verificar permisos sobre configuración y plantillas. |
| 4 | Confirmar disponibilidad de herramientas avanzadas. |

## 4.2.4 Gestión de usuarios y permisos editoriales

| Acción | Instrucción |
| --- | --- |
| Crear editor | Asignar usuario con permisos de creación de contenido. |
| Definir flujo de aprobación | Configurar estados (borrador, pendiente, aprobado, publicado). |
| Restringir acceso por sección | Otorgar permisos granulares por tipo de contenido. |
| Auditar cambios de usuario | Revisar histórico de ediciones por persona. |

## 4.2.5 Administración de taxonomía y metadatos

| Acción | Instrucción |
| --- | --- |
| Crear categoría | Definir jerarquía y descripción de nuevas secciones. |
| Gestionar etiquetas | Añadir, modificar o deprecar etiquetas de clasificación. |
| Configurar campos personalizados | Crear metadatos específicos del municipio o actuaciones. |
| Mapear compatibilidad | Sincronizar taxonomía con otros sistemas (web, app). |

## 4.2.6 Gestión de plantillas y estructura

| Acción | Instrucción |
| --- | --- |
| Crear plantilla de contenido | Definir campos obligatorios y estructura visual. |
| Aplicar estilos | Configurar formato de presentación y branding. |
| Versionado de plantillas | Mantener histórico de cambios para compatibilidad. |
| Validar integridad | Revisar que cambios no afecten contenido publicado. |

## 4.2.7 Auditoría y cumplimiento

| Acción | Instrucción |
| --- | --- |
| Revisar historial de cambios | Acceder a log completo de ediciones por contenido. |
| Validar cumplimiento normativo | Verificar que contenidos cumplan requisitos de accesibilidad. |
| Exportar informe de cambios | Descargar auditoría de modificaciones por período. |
| Gestionar versiones | Restaurar versiones anteriores si es necesario. |

## 4.2.8 Sincronización con otros sistemas

| Acción | Instrucción |
| --- | --- |
| Configurar conexión a web | Validar que cambios en CMS se reflejen en web pública. |
| Sincronizar con aplicación móvil | Configurar distribución de contenido a apps. |
| Configurar feed de datos | Mapear contenidos a otros canales de difusión. |
| Validar propagación | Verificar que todos los canales reciben actualizaciones. |

## 4.2.9 Gestión de incidencias administrativas

| Incidencia | Actuación recomendada |
| --- | --- |
| Contenido no se publica | Verificar estado de aprobación y permisos del publicador. |
| Cambios no se sincronizan | Revisar estado de conexión con sistemas de destino. |
| Historial incompleto | Comprobar configuración de auditoría y retención de datos. |
| Acceso denegado | Validar roles y permisos en el módulo administrativo. |

## 4.2.10 Buenas prácticas administrativas

1. Establecer flujos de aprobación claros según niveles de autoridad.
2. Documentar cambios en taxonomía antes de implementar.
3. Realizar backups antes de cambios estructurales significativos.
4. Revisar periódicamente el uso de plantillas y campos personalizados.
5. Mantener coherencia entre taxonomía de CMS y otros sistemas.
