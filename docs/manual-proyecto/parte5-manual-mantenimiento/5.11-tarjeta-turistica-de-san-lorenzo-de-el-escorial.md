# 5.11 Actuación 11: Tarjeta Turística de San Lorenzo de El Escorial - Mantenimiento

## 5.11.1 Objeto del manual de mantenimiento

La presente sección describe los procedimientos de mantenimiento preventivo y correctivo de
la Tarjeta Turística, incluyendo auditoría de integridad de datos, validación de
transacciones y gestión de incidencias operacionales.

## 5.11.2 Equipamiento y componentes sujetos a mantenimiento

| Componente | Tipo de mantenimiento | Frecuencia |
| --- | --- | --- |
| Base de datos de tarjetas | Integridad, backup, optimización | Diaria |
| Sistema de validación en puntos | Sincronización, auditoría de acceso | Semanal |
| Servidores de aplicación | Rendimiento, monitorización | Diaria |
| Conexión con puntos colaboradores | Validación de integración | Semanal |
| Registro de transacciones | Integridad, auditoría | Diaria |

## 5.11.3 Plan de mantenimiento preventivo

| Tarea | Descripción | Frecuencia | Responsable |
| --- | --- | --- | --- |
| Validación de integridad | Verificar que datos de tarjetas son consistentes | Diaria | DBA |
| Auditoría de transacciones | Revisar descuentos aplicados recientemente | Diaria | Auditor |
| Test de conectividad con puntos | Validar que puntos pueden validar beneficios | Semanal | Equipo técnico |
| Auditoría de acceso | Revisar quién accedió a qué datos | Semanal | Auditor seguridad |
| Backup de base de datos | Realizar copia de seguridad completa | Diaria | Automatizado |

## 5.11.4 Procedimiento de validación de integridad

| Paso | Instrucción |
| --- | --- |
| 1 | Ejecutar consulta de integridad de base de datos. |
| 2 | Validar que no hay registros huérfanos. |
| 3 | Verificar que saldos de tarjetas son coherentes. |
| 4 | Auditar transacciones sin contraparte. |
| 5 | Investigar discrepancias identificadas. |
| 6 | Aplicar correcciones si es necesario. |
| 7 | Documentar resultado de validación. |

## 5.11.5 Auditoría de transacciones

| Tarea | Descripción | Frecuencia |
| --- | --- | --- |
| Revisión diaria | Auditar descuentos aplicados en último día | Diaria |
| Análisis de patrones | Identificar transacciones anómalas | Semanal |
| Validación de montos | Verificar que descuentos corresponden a beneficios | Diaria |
| Auditoría de punto colaborador | Revisar transacciones por punto | Mensual |

## 5.11.6 Mantenimiento de conectividad con puntos

| Acción | Descripción | Frecuencia |
| --- | --- | --- |
| Test de validación | Simular validación de tarjeta en punto | Semanal |
| Monitorización de latencia | Revisar tiempo de respuesta de validación | Semanal |
| Auditoría de acceso | Verificar que punto solo accede datos autorizados | Trimestral |
| Validación de certificados | Verificar que punto tiene certificado válido | Mensual |

## 5.11.7 Gestión de incidencias operacionales

| Tipo de incidencia | Diagnóstico | Acción correctiva |
| --- | --- | --- |
| Tarjeta no valida | Revisar si tarjeta está activa en sistema | Activar tarjeta o investigar bloqueo |
| Descuento no se aplica | Auditar que beneficio está vigente | Reactivar beneficio o validar elegibilidad |
| Transacción duplicada | Revisar historial de transacciones | Invertir transacción duplicada |
| Punto no puede validar | Test de conectividad con punto colaborador | Reiniciar sincronización o revisar certificado |

## 5.11.8 Procedimiento de recuperación ante corrupción

| Paso | Instrucción |
| --- | --- |
| 1 | Detectar corrupción mediante validación de integridad. |
| 2 | Aislar sistema y notificar a administrador. |
| 3 | Restaurar desde backup más reciente validado. |
| 4 | Validar integridad de datos restaurados. |
| 5 | Reconciliar transacciones post-backup si es posible. |
| 6 | Reanudar operación normal. |
| 7 | Investigar causa de corrupción. |

## 5.11.9 Documentación de mantenimiento

| Documento | Contenido | Actualización |
| --- | --- | --- |
| Log de auditoría | Transacciones revisadas, fecha, resultado | Diaria |
| Registro de incidencias | Problemas identificados y resoluciones | Continua |
| Matriz de conectividad | Estado de conexión con cada punto colaborador | Semanal |
| Informe de compliance | Validación de cumplimiento regulatorio | Mensual |

## 5.11.10 Buenas prácticas de mantenimiento

1. Ejecutar auditoría diaria de transacciones para detectar anomalías.
2. Mantener validación de integridad automatizada con alertas.
3. Documentar todas las correcciones aplicadas en base de datos.
4. Realizar backup diario con pruebas de restauración mensuales.
5. Coordinar mantenimiento con responsables de puntos colaboradores.
