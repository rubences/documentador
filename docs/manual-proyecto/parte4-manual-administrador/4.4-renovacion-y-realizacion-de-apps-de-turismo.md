# 4.4 Actuación 4: Renovación y Realización de Apps de Turismo - Administración

## 4.4.1 Objeto del manual de administración

La presente sección describe las tareas de administración de las aplicaciones móviles de
turismo, incluyendo gestión de versiones, distribución, análisis de uso y gestión de
incidencias.

## 4.4.2 Perfiles de administración

| Perfil | Responsabilidades |
| --- | --- |
| Administrador de apps | Configuración técnica, versiones y distribución |
| Responsable de contenido app | Gestión de datos y actualizaciones dentro de la aplicación |
| Analista de uso | Seguimiento de descarga, instalación y comportamiento de usuarios |

## 4.4.3 Acceso al panel de administración de apps

| Paso | Instrucción |
| --- | --- |
| 1 | Acceder a la consola de gestión de aplicaciones móviles. |
| 2 | Iniciar sesión con credenciales de administrador. |
| 3 | Verificar acceso a tiendas de apps (iOS, Android). |
| 4 | Confirmar permisos para gestión de versiones y distribución. |

## 4.4.4 Gestión de versiones y actualizaciones

| Acción | Instrucción |
| --- | --- |
| Compilar nueva versión | Ejecutar build desde código fuente validado. |
| Realizar testing | Probar funcionamiento en dispositivos reales de cada plataforma. |
| Configurar versión para distribución | Asignar número de versión, notas de cambio y fecha de lanzamiento. |
| Distribuir en tiendas | Enviar a App Store e Google Play con cambios de instalador. |

## 4.4.5 Gestión de contenidos dentro de la app

| Acción | Instrucción |
| --- | --- |
| Actualizar datos turísticos | Sincronizar con CMS contenido que se distribuye en la app. |
| Gestionar mapas y localizaciones | Actualizar puntos de interés y coordenadas geográficas. |
| Configionar rutas sugeridas | Crear y modificar itinerarios recomendados. |
| Validar integridad de datos | Verificar que todos los cambios se reflejan correctamente. |

## 4.4.6 Distribución y disponibilidad

| Acción | Instrucción |
| --- | --- |
| Configurar compatibilidad de versión | Definir versiones mínimas de OS y dispositivos soportados. |
| Gestionar acceso regional | Configurar disponibilidad por región si aplica. |
| Activar lanzamiento gradual | Realizar rollout progresivo para usuarios como control de calidad. |
| Monitorizar descarga | Seguimiento de instalaciones por día y ubicación. |

## 4.4.7 Analítica y comportamiento de usuarios

| Acción | Instrucción |
| --- | --- |
| Configurar seguimiento de eventos | Rastrear acciones clave del usuario dentro de la app. |
| Analizar retención | Calcular qué porcentaje de usuarios regresa a la app. |
| Identificar caídas | Detectar pantallas donde usuarios abandonan la aplicación. |
| Generar informes de uso | Crear reportes de descarga, sesiones y características más usadas. |

## 4.4.8 Notificaciones push y comunicación

| Acción | Instrucción |
| --- | --- |
| Configurar servicio de push | Habilitar capacidad de enviar notificaciones a usuarios. |
| Definir segmentación | Crear grupos de usuarios para campañas dirigidas. |
| Programar campañas | Planificar envío de notificaciones para eventos específicos. |
| Medir efectividad | Revisar tasas de apertura y conversión de push. |

## 4.4.9 Gestión de incidencias administrativas

| Incidencia | Actuación recomendada |
| --- | --- |
| App no aparece en tienda | Revisar estado de revisión y requisitos de política de la plataforma. |
| Versión rechazada por tienda | Analizar motivo de rechazo y aplicar correcciones. |
| Datos no se actualizan | Verificar sincronización con CMS o API de contenido. |
| Alto número de desinstalaciones | Revisar problemas reportados y lanzar versión de corrección. |

## 4.4.10 Buenas prácticas administrativas

1. Mantener versionado coherente entre iOS y Android.
2. Realizar testing exhaustivo antes de distribución pública.
3. Documentar cambios en notas de versión claras para usuarios.
4. Monitorizar analítica regularmente para identificar problemas.
5. Mantener compatibilidad con versiones antiguas mientras sea viable.
