# 4.5 Actuación 5: Cartelería Digital Centralizada - Administración

## 4.5.1 Objeto del manual de administración

La presente sección describe las tareas de administración técnica del sistema de cartelería
digital centralizada, incluyendo gestión de equipamiento, configuración de canales y
monitorización de estado.

## 4.5.2 Perfiles de administración

| Perfil | Responsabilidades |
| --- | --- |
| Administrador de cartelería | Gestión de paneles, conectividad y distribución |
| Responsable técnico | Configuración de hardware y resolución de incidencias |
| Auditor de difusión | Seguimiento de campañas y efectividad |

## 4.5.3 Acceso al módulo administrativo de cartelería

| Paso | Instrucción |
| --- | --- |
| 1 | Acceder a la consola de gestión centralizada de cartelería. |
| 2 | Iniciar sesión con credenciales de administrador. |
| 3 | Verificar acceso a configuración de dispositivos e inventario. |
| 4 | Confirmar disponibilidad de herramientas de diagnóstico. |

## 4.5.4 Gestión del inventario de paneles

| Acción | Instrucción |
| --- | --- |
| Registrar nuevo panel | Documentar ubicación, modelo, resolución y conectividad. |
| Configurar parámetros del panel | Definir idioma, zona horaria, brillo y opciones de ahorro energético. |
| Asignar grupos de paneles | Crear conjuntos para distribuir contenido a múltiples ubicaciones. |
| Actualizar inventario | Mantener registro de cambios, traslados o retirada de equipamiento. |

## 4.5.5 Configuración de canales de distribución

| Acción | Instrucción |
| --- | --- |
| Crear canal de contenido | Definir agrupación de paneles que reciben la misma difusión. |
| Establecer permisos de publicación | Asignar quién puede crear campañas para cada canal. |
| Configurar horarios de funcionamiento | Definir horas de operación de paneles por ubicación. |
| Validar alcance | Verificar que el contenido llega a todos los paneles asignados. |

## 4.5.6 Monitorización y diagnóstico

| Acción | Instrucción |
| --- | --- |
| Revisar estado de conectividad | Verificar estado online/offline de cada panel en tiempo real. |
| Validar versión de firmware | Comprobar que el software del panel está actualizado. |
| Revisar ocupación de almacenamiento | Monitorizar espacio disponible para contenido. |
| Ejecutar pruebas de comunicación | Forzar actualización remota para verificar conectividad. |

## 4.5.7 Actualización y mantenimiento de equipamiento

| Acción | Instrucción |
| --- | --- |
| Descargar actualización de firmware | Obtener versión más reciente desde repositorio de proveedor. |
| Programar actualización de paneles | Agendar en ventana de baja demanda y horarios permitidos. |
| Ejecutar actualización remota | Distribuir nuevo firmware sin intervención in-situ. |
| Validar compatibilidad post-actualización | Verificar que todas las campañas activas funcionan correctamente. |

## 4.5.8 Auditoría y reportes

| Acción | Instrucción |
| --- | --- |
| Revisar historial de cambios | Acceder a log de campaigns publicadas, modificadas o eliminadas. |
| Generar informe de disponibilidad | Calcular uptime de cada panel y grupo. |
| Exportar métricas de difusión | Obtener datos de tiempo en pantalla por campaña. |
| Analizar efectividad | Evaluar impacto de campañas según parámetros disponibles. |

## 4.5.9 Gestión de incidencias administrativas

| Incidencia | Actuación recomendada |
| --- | --- |
| Panel offline prolongado | Contactar con soporte técnico o revisar alimentación eléctrica. |
| Contenido no se actualiza en panel | Forzar sincronización remota y validar conectividad de red. |
| Problema de visualización | Revisar configuración de resolución y validar archivo multimedia. |
| Múltiples paneles afectados | Verificar estado de infraestructura de distribución central. |

## 4.5.10 Buenas prácticas administrativas

1. Realizar inventario mensual de paneles operativos y estado de equipamiento.
2. Documentar todas las actualizaciones de firmware y fecha de aplicación.
3. Ejecutar pruebas periódicas de carga en red de distribución.
4. Mantener política de respaldo para actualización de firmware.
5. Revisar logs de sincronización regularmente para identificar patrones de fallos.
