# 4.7 Actuación 7: Monitorización de Flujos Turísticos - Administración

## 4.7.1 Objeto del manual de administración

La presente sección describe las tareas de administración técnica del sistema de
monitorización de flujos, incluyendo configuración de nodos, definición de alertas y
explotación de datos históricos.

## 4.7.2 Perfiles de administración

| Perfil | Responsabilidades |
| --- | --- |
| Administrador del sistema de aforos | Configuración de nodos y zonas de conteo |
| Responsable de alertas | Definición de umbrales y políticas de notificación |
| Analista de datos | Explotación histórica y generación de reportes |

## 4.7.3 Acceso al módulo administrativo de aforos

| Paso | Instrucción |
| --- | --- |
| 1 | Acceder a la consola de administración del sistema de monitorización. |
| 2 | Iniciar sesión con credenciales de administrador. |
| 3 | Verificar acceso a configuración de nodos y parámetros de alerta. |
| 4 | Confirmar disponibilidad de herramientas de análisis histórico. |

## 4.7.4 Gestión de nodos y sensores

| Acción | Instrucción |
| --- | --- |
| Registrar nuevo nodo | Documentar ubicación, tipo de sensor y capacidad de conteo. |
| Configurar zona de detección | Definir área geográfica monitoreada por nodo. |
| Calibrar sensor | Ajustar parámetros de sensibilidad según configuración física. |
| Validar comunicación | Verificar conectividad y latencia de transmisión de datos. |

## 4.7.5 Configuración de flujos y zonas

| Acción | Instrucción |
| --- | --- |
| Definir flujo de entrada | Configurar dirección y sentido de conteo. |
| Definir flujo de salida | Establecer vector opuesto para movimiento bidireccional. |
| Agrupar nodos por zona | Crear agrupación de sensores por área turística. |
| Validar integridad de conteo | Verificar que entradas y salidas son consistentes. |

## 4.7.6 Gestión de alertas y notificaciones

| Acción | Instrucción |
| --- | --- |
| Crear regla de alerta | Definir condición de umbral que dispara notificación. |
| Configurar canales de notificación | Establ ecer SMS, correo, push o sistemas de alerta específicos. |
| Definir escalado de alertas | Crear protocolo de escala si alerta persiste. |
| Auditar activación de alertas | Revisar histórico de disparos y respuestas operativas. |

## 4.7.7 Configuración de retención de datos

| Acción | Instrucción |
| --- | --- |
| Definir período de almacenamiento | Establecer cuánto tiempo se conservan datos sin procesar. |
| Configurar agregación horaria | Establecer consolidación automática de datos por período. |
| Configurar archivado | Mover datos históricos a almacenamiento de largo plazo. |
| Gestionar cuotas de almacenamiento | Monitorizar uso de espacio en base de datos. |

## 4.7.8 Análisis y reporting

| Acción | Instrucción |
| --- | --- |
| Generar informe de aforo | Crear resumen de ocupación por período y ubicación. |
| Extraer series históricas | Obtener datos granulares para análisis comparativo. |
| Calcular indicadores | Computar tasa de ocupación, picos, valles y estadísticas. |
| Exportar datos para BI | Alimentar herramientas de Business Intelligence con información. |

## 4.7.9 Validación y calidad de datos

| Acción | Instrucción |
| --- | --- |
| Revisar consistencia de datos | Verificar que conteos son coherentes (entradas ≈ salidas). |
| Identificar anomalías | Detectar conteos anormales o patrones sospechosos. |
| Aplicar correcciones | Ajustar valores identificados como errados tras validación manual. |
| Documentar correcciones | Registrar cambios realizados y razón técnica. |

## 4.7.10 Gestión de incidencias administrativas

| Incidencia | Actuación recomendada |
| --- | --- |
| Un nodo no reporta datos | Revisar conectividad de red y estado del sensor. |
| Conteo anómalamente alto | Validar en campo y revisar configuración de detección. |
| Alerta no se dispara | Verificar condición de alerta y canal de notificación. |
| Datos históricos incompletos | Revisar período de retención y ejecutar recuperación si procede. |

## 4.7.11 Buenas prácticas administrativas

1. Realizar auditoría mensual de precisión de conteos por nodo.
2. Documentar cambios en configuración de zonas o umbrales.
3. Mantener backup de configuración de alertas.
4. Revisar periódicamente coherencia de datos en período de validación.
5. Realizar validación in-situ de funcionamiento de nodos mensualmente.
